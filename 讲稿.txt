
**视频标题**
FinGraph：一个面向美股宏观分析的个人金融情报图谱

**00:00 - 00:30 开场**
大家好，这个视频我想介绍一下我最近搭建的一个个人金融情报系统，叫做 FinGraph。

首先，整个项目是围绕美股市场搭建的，FinGraph 关注的不是泛泛而谈的财经新闻，而是尽量把所有信息都放回到“这些变量会如何影响美股定价、风险偏好、企业盈利和宏观环境”这个核心问题上。

对于刚入门的小白来说，很多经济学概念、金融名词和市场变量其实并不是孤立难懂，而是缺少一个清晰的思维框架。比如利率、通胀、美元、财政、产业、企业盈利、地缘风险和市场情绪之间到底是什么关系，往往在脑海里并没有形成一套层级化的分析路径。

所以很多时候我们会去问 AI，但问题是，AI 并不知道我们的金融基础到底有多薄弱，也不知道我们真正卡住的是哪个层级、哪个概念、哪条逻辑链。更现实的是，很多新手甚至不知道应该怎么向 AI 提问，应该从哪个角度切入，也不知道一个完整的金融分析到底应该覆盖哪些方面。

简单来说，FinGraph 想解决的问题是：每天市场上信息非常多，但真正影响美股定价的变量其实可以拆成几个层级，比如货币、央行、财政、产业、企业、地缘、社会和市场情绪。这个网站就是把这些信息源自动收集、分类、链接，并最终导出成一个可以交给大模型分析的 Skill Pack 或 Prompt。

**00:30 - 01:10 顶部与整体结构**
现在我们看到的是首页，也就是“美股金融分析图谱”。

顶部这里会显示当前的数据模式，比如是否已经连接 Supabase 实时数据库，以及当前数据生成时间。下面这一行会显示自动抓取数据的计划时间，也就是纽约时间每天中午 12 点执行，同时也会显示访问者当前时区和纽约时区的实时时间。

整个页面采用三栏布局。中间是最重要的主工作区，左右两侧是辅助分析区。所有面板都可以拖拽排序、折叠展开，也可以切换不同的主题色。这样我可以根据自己的阅读习惯，把最常用的模块放到最上面。

**01:10 - 01:55 左侧栏：摘要与宏观脉络**
先看左侧栏。

第一个面板是“今日一句话总结”。它会把当天收集到的宏观、政策和市场信息压缩成一个简短判断，帮助我快速进入状态。这里不会把它当成投资建议，而是当成一个当天市场环境的摘要。

下面是“八个分析层健康度评分”。这里把金融系统拆成几个层级，例如货币层、央行层、财政层、产业层、企业层、地缘层、社会层和市场层。每个层级会有一个分数和状态，用来帮助判断当前压力主要集中在哪里。

再往下是“市场概览”和“利率与通胀趋势”。这些面板主要展示真实 API 或市场数据，比如利率、通胀、ETF、能源价格等指标，并且每个指标都会保留来源链接，方便点击回到原始页面检查。

后面还有“通胀分项监测”“政策预期与央行证据”“财政与社会压力”。这些模块分别对应美股定价里非常关键的几个变量：通胀是否反弹、央行是否偏鹰、财政赤字和债务压力是否影响长端利率。

**01:55 - 02:50 中间栏：主工作区**
中间这一栏是核心区域。

最上方是 TradingView 图表入口。这里嵌入了 SPY、QQQ、TLT、GLD、USO、XLK、XLF、XLE、NVDA 等常用资产或板块图表。点击不同标签，就可以快速查看对应图表，不需要每次重新打开 TradingView。

下面是“外部信息终端”。这个模块把一些外部财经信息源直接嵌入进来，比如 TradingView 经济日历、美股热力图、市场概览、市场新闻，还有金十数据、FinancialJuice、财联社电报、东方财富快讯和 Investing.com 日历。这样我可以在一个页面里同时查看中文快讯、英文市场快讯、宏观日历和市场热力图。

再往下是“九层结构图”。这个面板展示的是 FinGraph 背后的分析框架。它不是简单罗列新闻，而是把外部冲击、社会结构、财政债务、央行政策、货币体系、产业结构、企业盈利、市场情绪和最终资产定价串联起来。

接着是“关键影响矩阵”。这个面板会把当天事件映射到不同层级，帮助我判断一个事件到底影响的是央行、财政、产业，还是市场风险偏好。

最底部是“Skill Pack 导出”。这是整个项目最重要的输出功能。用户可以选择导出 ZIP 包，或者导出单个 TXT 文件，也可以直接复制到剪切板。导出的内容包括完整的金融分析方法论、知识图谱背景、近期真实事件、指标、来源链接和默认 Prompt。也就是说，我可以把这个文件直接交给另一个大模型，让它基于完整上下文生成一份宏观或美股分析报告。

**02:50 - 03:35 右侧栏：资讯、风险与事件**
右侧栏主要放事件和风险类信息。

最上面是“今日重要资讯与规则解读”。这里会列出当天比较重要的事件，并且每条事件都会标注来源、强度、周期、相关层级和原始链接。这个设计的重点是可追溯，任何信息都不应该只停留在摘要层面，而是可以点击回到原始网页。

下面是“地缘热点事件源”。这个模块主要用于捕捉地缘冲突、制裁、能源安全、航运风险、灾害和供应链相关事件。因为这些事件经常会影响油价、美元、通胀预期和风险资产估值。

再往下是“风险仪表盘”“收益率曲线”“期货仓位 / COT”和“企业财报日历”。这些面板分别对应市场风险、利率结构、机构仓位和企业盈利周期。它们可以帮助我判断市场当前到底是在交易宏观、交易流动性，还是交易企业盈利。

**03:35 - 04:10 数据源与底部信息源总览**
页面底部是“信息源总览”。

这里分成两类。第一类是嵌入平台与网页入口，比如 TradingView、金十数据、FinancialJuice、财联社、东方财富和 Investing.com。每个卡片都有 logo、类型标签和简短说明，点击就可以打开原始网站。

第二类是 API、RSS 和数据库来源。这里列出了 FinGraph 当前接入或规划接入的数据源，比如 FRED、BLS、BEA、Federal Reserve、U.S. Treasury、SEC EDGAR、GDELT、World Bank、Stooq、EIA、CFTC、Alpha Vantage、Twelve Data 等。每个来源会显示它的类型、可靠性、更新频率、是否需要 API Key，以及是否已经接入。

这个区域的作用是让整个项目更加透明：我不希望页面里的数据看起来像黑箱，所有来源都应该可以被追踪和验证。

**04:10 - 04:40 其他页面说明**
除了首页之外，顶部还有几个子页面。

“图谱结构”页面主要展示九层分析框架和不同节点之间的关系，帮助理解 FinGraph 的底层分析逻辑。

“事件流”页面更像一个事件数据库，可以集中查看近期采集到的真实事件和原始链接。

“数据源”页面展示所有 API、RSS、数据库和外部来源的接入状态，方便检查哪些已经启用、哪些需要 Key、哪些还只是计划来源。

“Skill Pack”页面则是导出中心，可以单独生成适合交给 AI 的分析上下文。

**04:40 - 05:10 总结与使用方式**
总结一下，FinGraph 的使用方式大概是这样的：

每天系统定时抓取数据，写入 Supabase 数据库。用户打开网页后，可以先看一句话总结和健康度评分，然后查看 TradingView 图表、外部快讯、关键事件和风险面板。如果需要深入分析，就可以导出 Skill Pack，把完整上下文交给大模型，让它生成更系统的美股宏观分析报告。

这个项目的重点不是预测明天涨跌，而是建立一个稳定的信息工作流：收集真实来源、保留原始链接、按金融层级组织信息、最后导出成可分析的结构化上下文。

这就是 FinGraph 当前的整体功能。后续还可以继续扩展更多数据源、更多可视化面板，以及更细的资产和行业跟踪。

最后需要说明的是，因为这个网站里嵌入了一些海外财经平台和外部信息源，所以部分面板在不同网络环境下可能需要使用科学上网工具才能正常加载。如果某些 iframe 页面暂时打不开，也可以直接点击对应面板里的“打开源页面”，跳转到原始网站查看。

另外，如果有观众想基于我的这个项目框架继续魔改，比如替换数据源、增加新的分析层级、接入自己的 API，或者改成适合自己投资风格的金融情报系统，也可以点击页面左下角的 GitHub 仓库入口。整个项目的源代码已经开源，你可以把代码交给自己的 AI Agent 阅读，让它帮你理解项目结构、配置环境变量，并一步步指导你完成部署。
